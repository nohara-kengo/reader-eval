# 受け入れ条件 — Issue #31（M365 Entra ID SSO + 空ダッシュボード）

## 想定シナリオ

1 受け入れ条件 = 1 テスト。本 Issue は「未認証は弾く / SSO で入れる / 入ったら空ダッシュボード / 抜けられる」を最小で通す。

> **検証区分の注記**: 実 Microsoft への SSO ラウンドトリップ（実ログイン）は **Entra アプリ登録と実資格情報が必要**で、ローカル / CI では実施しない。下表で **[要資格情報]** はステージング等で人手確認、**[資格情報不要]** はモック / ガード挙動 / ビルドで自動検証する。

## 人手確認項目

- [ ] [資格情報不要] 未認証で `/dashboard` にアクセスすると `/login` にリダイレクトされる
- [ ] [資格情報不要] `/login` に「Microsoft でサインイン」導線が表示される
- [ ] [要資格情報] Entra ID でサインイン成功後 `/dashboard` に遷移し、空のダッシュボード（プレースホルダ）が表示される
- [ ] [要資格情報] サインアウトでセッションが破棄され、再度 `/dashboard` で `/login` に戻る
- [ ] [要資格情報] サインインをキャンセル / 失敗すると `/login` にエラー表示（技術詳細なし）で戻る
- [ ] [資格情報不要] レスポンス / HTML / クライアントバンドルにシークレットが含まれない（`NEXT_PUBLIC_` 混入なし）
